# Правила визуализации алгоритмов (статьи / сайдбар)

Документ описывает единый подход к анимациям в учебных статьях. Контент статей хранится в `db.json`; блоки с `"type": "animation"` рендерятся как **iframe** через `ContentRenderer`.

## Инфраструктура

1. **Базовые стили** подключаются автоматически: `front/src/shared/ui/content/viz-animation-base.css` (через `VIZ_ANIMATION_BASE_CSS` в `content-renderer.tsx`).
2. **Разметка iframe** всегда оборачивается в:
   - `.viz-viewport` — внешний контейнер: **центрирование**, `overflow: hidden`, без скролла.
   - `.viz-stage` — внутренняя сцена: **максимум площади**, `flex`, укладка контента.
3. **Вертикально «высокие» схемы** (стек, глубокая рекурсия): в блоке анимации задать `"vizLayout": "tall"` — включается режим колонки внутри viewport (без скролла; контент должен **умещаться** за счёт `clamp`, меньшего числа шагов или компактной сетки).
4. **Перезапуск**: кнопка поверх iframe — `GlassButton` (круглая), см. `content-renderer.module.css` → `.animationReplayGlass`.

## Визуальный язык

- **Форма элементов данных** — прямоугольники с **скруглением** (класс `.viz-cell`), крупные, на всю доступную ширину ряда (`flex: 1 1 0`), высота через `clamp`, чтобы заполнять область и не вылезать.
- **Цвета** — только из палитры CSS-переменных в `viz-animation-base.css`:
  - фон ячейки: `--viz-surface`, граница `--viz-border`;
  - акцент текущего шага: `.viz-cell--active` (`--viz-accent`);
  - сравнение / «кандидат»: `.viz-cell--compare` (`--viz-warn`);
  - успех / база: `.viz-cell--success` (`--viz-success`);
  - подписи: `.viz-caption`, `.viz-hint`, `.viz-badge`.
- **Не вводить** произвольные hex-цвета в `css` блока, кроме редких исключений (лучше расширить базу, а не статью).

## Что показываем (и что нет)

- **Нет «консоли»**: не имитировать терминал, списки логов, прокручиваемые `logLine` и т.п.
- **Да** визуальные состояния структуры данных: массив, стек, очередь, таблица DP, этапы divide-and-conquer.
- **Да** короткие текстовые пояснения **одной строкой** (или 2 коротких): `.viz-hint` и/или `.viz-badge` — это *подпись к кадру*, а не журнал событий.
- **Псевдокод / формулы** — в отдельных блоках `code` статьи, не дублировать длинным текстом внутри iframe.

## Компоновка

- Горизонтальные ряды: `.viz-row` + набор `.viz-cell`.
- Вертикальный стек: `.viz-stack-v` + `.viz-cell`.
- Две зоны (например, стек + статус): `.viz-split` + `.viz-panel`.
- Группы (подмассивы при merge sort): `.viz-group` с вложенными `.viz-cell`.
- Столбчатая диаграмма (рост n): `.viz-bars` + `.viz-bar`.
- Сетка DP: `.viz-grid` + `.viz-cell`.

## Анимация и укладка

- **Без скролла** внутри iframe: `html, body { overflow: hidden }` уже в базе.
- Если контент «не влезает»:
  - уменьшить число элементов на экране;
  - увеличить `"height"` блока анимации в `db.json`;
  - использовать `"vizLayout": "tall"` и вертикальную компоновку;
  - укрупнить шаги (меньше промежуточных кадров).
- Тайминги: `300–600 ms` на шаг, без бесконечных `setInterval` без сброса при перезапуске — скрипт должен корректно отрабатывать при повторном выполнении (новый iframe при нажатии ▶).

## Соответствие блока в `db.json`

Минимальный шаблон:

```json
{
  "id": "anim-example",
  "type": "animation",
  "html": "<div class=\"viz-col\">...</div>",
  "css": "",
  "js": "(function(){ ... })();",
  "width": "100%",
  "height": 240,
  "vizLayout": "default",
  "showPlayButton": true
}
```

- `html` — только **внутреннее** содержимое сцены (без собственного `html/body`); обёртки `.viz-viewport` / `.viz-stage` добавляет рендерер.
- `css` — точечные дополнения; основная типографика и палитра — в базе.

## Ориентиры из практики визуализации (интернет / учебные ресурсы)

- Показывать **состояние** структуры данных в момент времени, а не поток текста.
- Выделять **один активный элемент** (или пару для сравнения).
- Для рекурсии — **дерево вызовов** или **стек кадров** + краткий статус «какой вызов сейчас».
- Для DP — **заполнение таблицы** с подсветкой текущей ячейки и уже посчитанных.
